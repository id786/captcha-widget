You can also add protected-con-1 to a button to hide it until the CAPTCHA is completed.

Understanding CAPTCHA IDs

The ID number (like '1' in the examples) connects the CAPTCHA container to the protected content and buttons. When CAPTCHA container 1 is completed, it will unlock all protected content and buttons with ID 1.

If you use different IDs (e.g., 9999) for the CAPTCHA container, you must use the same ID for the protected content and buttons. Completing captcha-container-9999 will unlock all buttons and content with ID 9999.

You can use multiple CAPTCHAs on the same page, but each must have a unique ID. For example:

  • First CAPTCHA: id="captcha-container-1" with protected elements: protected-btn-1 and protected-con-1
  • Second CAPTCHA: id="captcha-container-2" with protected elements: protected-btn-2 and protected-con-2

There's no limit to the number of CAPTCHAs you can use on a single page.

Note: The ID numbering system allows you to create complex protection scenarios where different CAPTCHAs unlock different parts of your website.

Important Implementation Notes

CRITICAL: Use specific inline styles for protected elements to ensure maximum security.

  • Add the disabled attribute to protected buttons
  • Add style="display:none;" to protected content

Why These Rules Are Important

Our CAPTCHA loader.js needs time to load, depending on your internet speed. Before the script loads, your protected buttons would be enabled and your protected content would be visible, which could create security vulnerabilities until the loader.js fully loads.

Implementation Examples

Your protected button code should look like this:

<button disabled class="protected-btn-1">DOWNLOAD</button>

Your protected content code should look like this:

<div class="protected-con-1" style="display:none;">Protected Content</div>
